================================================================================
SCORING SYSTEM
================================================================================

Overall Score = 
  config * 0.25 +
  usage * 0.25 +
  knowledge * 0.20 +
  reliability * 0.20 +
  security * 0.10

Urgency Mapping:
  0-29:   CRITICAL (immediate action required)
  30-49:  HIGH (major issues present)
  50-69:  MEDIUM (several improvements needed)
  70-100: LOW (healthy, routine monitoring)

Scoring Dimensions:

A. CONFIGURATION (25%)
   Checks: system prompt quality, flow definition, model parameters
   Penalties: missing name (-25), short prompt (-15), invalid temp (-20),
              no flow (-30), no content (-25), invalid duration (-10)

B. USAGE (25%)
   Checks: activity recency, conversation volume, error rates
   Penalties: >30 days inactive (-40), zero conversations (-30), no 7d activity
              (-25), <10% error rate (-20)

C. KNOWLEDGE BASE (20%)
   Checks: KB completeness, chunk count, embedding freshness
   Default (no KB): 50 points
   Penalties: zero chunks (-40), <5 chunks (-20), >90 days stale (-15)

D. RELIABILITY (20%)
   Checks: recent error rates, approval queue, eval coverage
   Penalties: >15% error rate (-30), >10 pending approvals (-20),
              no eval suites (-25), failed runs (-2 each)

E. SECURITY (10%)
   Checks: public exposure, ownership, hardcoded secrets
   Penalties: public (-30), no owner (-20), secrets (-15)
   Bonus: ECC enabled (+5)

================================================================================
METHODOLOGY HIGHLIGHTS
================================================================================

1. DETERMINISTIC SCORING
   All scores are calculated via explicit penalty formulas, not ML/heuristics.
   Fully reproducible; can trace any agent's score to specific violations.

2. PARALLEL DATA FETCHING
   7 queries run concurrently per agent:
   - Agent metadata + Flow
   - Knowledge base + chunks
   - Conversations stats
   - Analytics events
   - Human approvals
   - Eval suite health

3. BATCH PROCESSING
   Agents processed 10 at a time with asyncio.gather().
   Scales to 1000+ agents without memory overhead.

4. CONFIGURABLE THRESHOLDS
   All audit limits defined in AuditConfig class:
   - DAYS_INACTIVE_THRESHOLD = 30
   - CRITICAL_SCORE_THRESHOLD = 30
   - HIGH_SCORE_THRESHOLD = 50
   - MEDIUM_SCORE_THRESHOLD = 70
   Adjust to customize sensitivity.

5. COMPREHENSIVE FINDINGS
   Each agent receives list of specific issues detected:
   e.g., "No KB defined", "Last updated 45 days ago", "5% error rate"
   Actionable for remediation.
